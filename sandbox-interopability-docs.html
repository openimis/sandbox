How identity verification fits into enrolment

This page walks through the integration end to end: OpenID Connect and OAuth 2.0 on the wire, GraphQL between the React insuree module and Django, and how verified attributes land in the record.

Step 1: Open the insured person in openIMIS

Staff use the standard UI to search and prepare edits. The MOSIP path is an add-on that does not replace openIMIS permissions or business rules.

Protocols and technology

  • Frontend: React 17, Material UI v4, @openimis/fe-insuree, graphqlWithVariables.
  • API: Follows the openIMIS GraphQL schema to establish the insuree UUID and draft state.

Step 2: Redirect to MOSIP (eSignet)

MOSIP’s eSignet acts as an OpenID Provider (OIDC). After login, the browser returns to a redirect URI with an authorization code and state.

Implementation Details

  • Flow: response_type=code (Authorization Code Flow) with PKCE (RFC 7636).
  • Security: code_challenge and code_challenge_method=S256.
  • UI Surface: generateSignInUrl and MosipCallbackPage handle the return path.

Step 3: Server-side Token Exchange

The browser passes the code to openIMIS via GraphQL. The backend performs the exchange over HTTPS using a JWT client assertion signed with a private key.

Back-channel Protocols

  • Grant: grant_type=authorization_code via POST to token endpoint.
  • Auth: RFC 7523 jwt-bearer client assertion using PRIVATE_KEY_BASE64.
  • Backend: ExchangeCodeMutation (Graphene) using httpx.

Step 4: Call UserInfo with Access Token

The backend calls MOSIP’s UserInfo endpoint with Authorization: Bearer <token> to retrieve name-verified claims.

Claims Processing

  • Standard Claims: sub, name, birthdate, gender, email, phone_number, address, picture.
  • Handling: UserInfoMutation returns raw payload to fetchUserInfoRaw.

Step 5: Map claims to insuree model

The populateInsureeFromAuthCode orchestrator maps OIDC claims to openIMIS fields, including normalizing dates and converting pictures to Base64.

Persistence Logic

  • GraphQL: Uses existing create/update mutations from openimis-be-insuree_py.
  • UX: MosipCallbackPage tracks progress (loading/success/error) for the operator.

Security & Trust Boundaries

Server-side Exchange

Private JWKs and Client IDs stay in the server environment, never shipped to the React client.

Standards

OAuth 2.0 + PKCE, OIDC, RFC 7523, RS256, and GraphQL over HTTPS.

Operational Note

Redirect URIs and Client IDs must perfectly match the MOSIP eSignet registration.
